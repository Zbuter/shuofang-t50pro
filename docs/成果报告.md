## 二、技术实现概要

### 2.1 架构设计

采用四层分层架构：

```
应用层 (CLI/API/GUI)
    ↓
服务层 (PrintJob/Template/DeviceManager)
    ↓
驱动层 (TSPLCommand/ImageConverter/StatusParser)
    ↓
通信层 (USB/BLE/TCP Transport)
    ↓
硬件层 (T50Pro 打印机)
```

### 2.2 核心功能

#### 通信层 (Transport)
- **USBTransport**: 基于 pyserial 的 USB 虚拟串口通信
- **BLETransport**: 基于 bleak 的蓝牙 BLE 通信
- **TCPTransport**: 基于 socket 的网络通信
- 统一抽象接口，支持上下文管理器

#### 驱动层 (Driver)
- **TSPLCommand**: 完整的 TSPL 指令集生成器
  - 标签设置：SIZE, GAP, SPEED, DENSITY
  - 文本打印：TEXT（支持中英文字体）
  - 条形码：BARCODE（支持 9 种码制）
  - 二维码：QRCODE（4 种纠错级别）
  - 图像：BITMAP（自动转换）
  - 图形：BOX, LINE, REVERSE
  - 控制：PRINT, CALIBRATE, INIT

- **ImageConverter**: 图像处理工具
  - 位图转换（1bpp）
  - 阈值二值化
  - 尺寸调整
  - 旋转/反转

- **StatusParser**: 状态解析器
  - 字节响应解析
  - ASCII 响应解析
  - 错误码解码

#### 服务层 (Service)
- **PrintJob**: 打印任务构建器
  - 链式 API
  - 多元素支持
  - JSON 序列化

- **Template**: 模板系统
  - 字段定义
  - 数据绑定
  - 模板持久化

- **DeviceManager**: 设备管理
  - USB/BLE/网络扫描
  - 自动检测
  - 连接管理

## 四、使用示例

### 4.1 基础打印

```python
from printer_t50pro import Printer, PrintJob

# 连接打印机
printer = Printer.usb("/dev/ttyUSB0")
printer.connect()

# 创建标签
job = PrintJob(50, 30)
job.add_text("设备：交换机 A", 10, 10, size=2)
job.add_qrcode("SN:SW2024001", 10, 20)

# 打印
printer.print_job(job)
printer.disconnect()
```

### 4.2 模板打印

```python
from printer_t50pro import Template

# 定义模板
template = Template("device", 50, 30)
template.add_text_field("name", 10, 10, size=2)
template.add_qrcode_field("sn", 10, 20)

# 批量打印
devices = [
    {"name": "交换机 A", "sn": "SW001"},
    {"name": "交换机 B", "sn": "SW002"},
]

for device in devices:
    job = template.render(device)
    printer.print_job(job)
```

### 4.3 状态监控

```python
status = printer.get_status()
print(f"就绪：{status.is_ready}")
print(f"纸张：{status.paper_status}")
print(f"温度：{status.head_temp}°C")
```

## 六、已知限制

### 6.1 协议相关

1. **TSPL 指令集**: 基于通用 TSPL 规范，硕方可能有私有扩展指令
2. **状态查询**: 响应格式需根据实际设备调整
3. **蓝牙 UUID**: BLE 服务 UUID 需实际设备确认

### 6.2 功能相关

1. **切刀支持**: T50Pro 无切刀，未实现相关功能
2. **RFID**: 不支持 RFID 标签（如设备支持）
3. **固件升级**: 未实现固件升级功能

### 6.3 建议

1. 首次使用时用官方软件抓包验证指令格式
2. 根据实际设备响应调整状态解析逻辑
3. 蓝牙连接需确认正确的服务 UUID
